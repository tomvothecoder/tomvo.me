# Planning Template

Use this template for non-trivial tasks.

## Task

Describe the requested change in one to three sentences.

## Background

Summarize relevant context, current behavior, and why the change is needed.

## Goals

Define clear outcomes that indicate success.

- Goal 1
- Goal 2

## Constraints

Document requirements and restrictions.

- Preserve existing architecture and conventions
- Avoid unnecessary dependencies
- Avoid unintended breaking changes
- Add task-specific constraints as needed

## Relevant Files

List files, directories, or modules likely involved.

- `path/to/file-or-directory`

## Step-by-Step Implementation Plan

Define ordered, concrete, verifiable steps.

1. Step one
2. Step two
3. Step three

## Risks or Edge Cases

Capture assumptions, risks, and edge conditions.

- Risk 1
- Edge case 1

## Verification Strategy

Define how the change will be validated.

- Commands to run
- Tests to execute
- Manual checks
- Expected outcomes
